\subsection{Болевые точки авторов курсов}

Разработка корпоративного учебного курса представляет собой многоэтапный
процесс: анализ целевой аудитории, постановка учебных целей, структурирование
модулей, создание текстового и~мультимедийного контента, тестирование
и~актуализация материалов.
По~данным отраслевых исследований, средние трудозатраты на~разработку
одного часа обучения составляют от~40 до~200~часов работы методиста
в~зависимости от~сложности и~уровня мультимедийности~\cite{chapman_elearning_hours_2010}.

Ключевые болевые точки, выявленные в~ходе Customer~Development:
\begin{itemize}
  \item \textbf{Разрыв компетентностей}: эксперт предметной области,
    как правило, не~обладает педагогическими компетенциями
    для~самостоятельной упаковки знания в~учебный продукт,
    что требует привлечения внешнего методиста или~подрядчика;
  \item \textbf{Скорость устаревания}: корпоративный контент
    (регламенты, продуктовые описания) обновляется чаще,
    чем методисты успевают актуализировать курсы;
  \item \textbf{Стоимость}: разработка курса у~внешнего подрядчика
    составляет от~[PLACEHOLDER]~тыс.\ руб.\ в~среднем по~российскому
    рынку B2B-обучения~\cite{smart_ranking_elearning_2024}.
\end{itemize}

В~рамках исследования проведено [PLACEHOLDER]~глубинных B2B-интервью

по~методу Mom~Test~\cite{fitzpatrick_mom_test_2013}.
Полный гайд и~сводка инсайтов приведены в~приложении~А.

Ключевые результаты:

[PLACEHOLDER]\,\% участников подтвердили готовность использовать

\subsection{Рыночный контекст: TAM / SAM / SOM}

Глобальный рынок корпоративного обучения (TAM) оценивается
в~\$383~млрд в~2023~году с~прогнозируемым ростом до~\$605~млрд
к~2027~году (CAGR~$\approx$~12\,\%)~\cite{edtech_market_report_2024}.

Целевой сегмент (SAM)~--- российский рынок B2B-платформ для~корпоративного
обучения с~AI-функциональностью: по~оценке Smart~Ranking~\cite{smart_ranking_elearning_2024},
объём составляет [PLACEHOLDER]~млрд~руб.\ в~2024~году.

Реально достижимый рынок (SOM) для~Adapstory на~горизонте 3~лет~---
[PLACEHOLDER]~млрд~руб.\ при~целевой доле [PLACEHOLDER]\,\%.

\subsection{Прямые конкуренты}

Рынок AI-инструментов для~автогенерации курсов включает несколько
активных игроков:

\textbf{Coursebox~AI}~\cite{coursebox_ai_2024}~--- облачный сервис,
генерирующий структуру курса из~PDF/URL за~один промпт.
Поддерживает экспорт в~SCORM и~интеграцию с~LMS-платформами.
Ограничения: отсутствие научно обоснованной методологии проектирования,
слабая поддержка русского языка, все данные обрабатываются
на~серверах OpenAI (нет суверенитета данных).

\textbf{iSpring~AI}~--- российский вендор с~функцией AI-генерации
структуры курса на~основе темы или~загруженного документа.
Интегрирован с~экосистемой iSpring~Suite.
Ограничения: единственный промпт без~педагогической методологии,
ограниченные возможности кастомизации учебных целей.

\textbf{Synthesia}~--- AI-платформа для~создания видеокурсов
с~AI-аватарами; не~специализируется на~генерации структуры курса
из~документов заказчика.

\subsection{Смежные решения}

\textbf{Google~NotebookLM}~--- генерация AI-подкастов и~конспектов
из~загруженных документов; не~предоставляет структуру учебного курса
и~не~интегрируется с~LMS.

\textbf{Khanmigo~(Khan~Academy)}~--- AI-тьютор для~студентов;
ориентирован на~B2C, не~поддерживает корпоративные регламенты.

\subsection{Пробел между проблемой и существующими решениями}

Анализ конкурентного ландшафта показывает, что ни~одно из~существующих
решений не~закрывает одновременно три требования корпоративного B2B-заказчика:
(1)~научно обоснованная методология проектирования курса,
(2)~полный суверенитет данных (on-premise LLM),
(3)~механизм автоматической актуализации при~изменении источника.

\subsection{Unfair Advantage}

Некопируемое конкурентное преимущество ИИ-методиста формируется
из~сочетания трёх факторов:
методологической вариативности (ADDIE/SAM/Backward~Design),
полного суверенитета данных (Qwen~2.5 через Ollama, on-premise)
и~глубокой интеграции в~экосистему LMS~Adapstory~---
ни~один из~конкурентов не~обеспечивает все~три одновременно.

\subsection{Формулировка исследовательского вопроса}

На~основании анализа существующих продуктов и~научной литературы
сформулирован исследовательский вопрос:

\textit{«В~какой мере stateful мультиагентная архитектура с~вариативной
методологической моделью превосходит single-prompt подход
при~автоматизированном создании русскоязычных образовательных материалов
по~показателям faithfulness, Bloom's~coverage и~времени генерации?»}

\subsection{LLM для автоматической генерации образовательного контента}

Применение LLM для~генерации учебного контента активно исследуется
с~2022~года.
Kasneci~et~al.~\cite{kasneci_llm_education_2023} провели систематический
обзор возможностей и~ограничений LLM в~образовании,
выделив автоматическую генерацию учебных материалов как~одно
из~наиболее перспективных направлений.
Pardos~\cite{pardos_llm_instructional_2023}
и~Tack~et~al.~\cite{tack_aied_2022}
показывают, что GPT-4 способен генерировать учебные объяснения
уровня опытного тьютора, однако отмечают проблему согласованности
педагогических целей.

Ключевое ограничение существующих подходов~---
использование единственного промпта без~педагогической методологии:
сгенерированный контент технически корректен, но~не~структурирован
по~Bloom's-уровням и~не~следует принципам ADDIE или~SAM~\cite{addie_molenda_2003, allen_sam_2012}.

\subsection{Мультиагентные системы для генерации контента}

Мультиагентные LLM-системы разделяют сложную задачу
между~специализированными агентами, повышая качество итогового
результата по~сравнению с~one-shot подходом~\cite{wu_autogen_2023}.

Shinn~et~al.~\cite{shinn_reflexion_2023} предложили паттерн Reflexion,
при~котором агент рефлексирует над~собственным выводом и~итеративно
его~улучшает.
Wang~et~al.~\cite{wang_plan_and_execute_2023} описывают паттерн
Plan-and-Execute: отдельный агент-планировщик составляет план действий,
исполнители реализуют отдельные шаги.
Именно эти паттерны легли в~основу архитектуры ИИ-методиста
(глава~3).

\subsection{RAG в образовании}

Retrieval-Augmented~Generation~\cite{lewis_rag_2020} позволяет
обогащать контекст LLM релевантными фрагментами из~корпуса знаний,
снижая галлюцинации и~повышая faithfulness ответов.
Es~et~al.~\cite{es_ragas_2023} предложили фреймворк Ragas
для~автоматизированной оценки RAG-систем,
включающий метрики faithfulness, answer~relevancy и~context~precision,
которые используются в~настоящей работе как~основные количественные показатели.

\subsection{Генерация мультимедиа для обучения}

Автоматическая генерация иллюстраций для~учебного контента
исследуется в~контексте диффузионных моделей~\cite{flux_2024, dalle3_2023}.
Генерация AI-подкастов~--- относительно новое направление:
Google NotebookLM показал высокую пользовательскую удовлетворённость
при~синтезе диалоговых подкастов из~документов.
В~настоящей работе применяются FLUX.1-dev для~иллюстраций
и~Silero~TTS~\cite{silero_tts_2024} для~TTS-подкастов
как~on-premise альтернативы облачным провайдерам.

\subsection{Методологии проектирования курсов в контексте LLM}

ADDIE (Analyze, Design, Develop, Implement, Evaluate)~\cite{addie_molenda_2003}
является наиболее распространённой моделью разработки обучения (ISD).
SAM~\cite{allen_sam_2012} предлагает итеративный подход к~дизайну,
снижающий риск «анализного паралича».
Backward~Design~\cite{wiggins_backward_design_2005}
строит курс от~ожидаемых результатов к~содержанию.

В~настоящей работе все~три методологии реализованы как~переключаемые
системные промпты агента CourseStructureAgent,
что~позволяет адаптировать конвейер под~предпочтения заказчика
без~изменения архитектуры.

\subsection{Оценка качества AI-генерируемого контента}

Таксономия Блума~\cite{bloom_taxonomy_1956}~--- иерархическая модель
когнитивных целей обучения из~шести уровней (знание, понимание,
применение, анализ, синтез, оценка)~--- служит стандартом оценки
педагогического качества учебных целей.
Применение таксономии Блума как~метрики качества AI-генерируемого
контента предложено в~\cite{kasneci_llm_education_2023}.

\subsection{Идентификация исследовательской лакуны}

Проведённый двухуровневый обзор позволяет идентифицировать
следующие пробелы в~существующей литературе:

\begin{itemize}
  \item существующие работы по~LLM-генерации образовательного контента
    не~используют Bloom's~coverage как~количественную метрику качества
    при~сравнении архитектур;
  \item ни~одна из~рассмотренных работ не~исследует stateful~DAG
    мультиагентную архитектуру с~переключаемой педагогической методологией
    для~генерации курсов;
  \item отсутствуют воспроизводимые бенчмарки с~русскоязычным
    образовательным корпусом для~сравнения RAG-конфигураций
    в~задаче генерации учебного контента.
\end{itemize}

Настоящая работа закрывает эти три пробела.

\subsection{Задача исследования}

На~основании анализа предметной области задача исследования формулируется
следующим образом: \textit{разработать мультиагентную систему генерации
структуры и~контента учебных курсов, обеспечивающую статистически значимое
превосходство над~single-prompt baseline по~педагогическим метрикам
(faithfulness, Bloom's~coverage) при~приемлемой unit-стоимости на~курс}.

\subsection{Методологическая основа}

Бизнес-требования формализованы в~терминах методологии
Customer~Value~Management (Amazon)~\cite{amazon_cvm_2014}:
задача ставится от~пользователя назад к~техническому решению
(Working Backwards).
Требования к~учебным целям определяются таксономией Блума~\cite{bloom_taxonomy_1956};
Jobs~To~Be~Done~\cite{christensen_jtbd_2016} использовалась
как~аналитическая рамка при~интерпретации результатов Customer~Development.

\subsection{Три пайплайна ИИ-методиста}

Система поддерживает три сценария применения:
\begin{enumerate}
  \item \textbf{Онбординг-курс}~--- входной документ: продуктовое описание
    или~бриф; методология~ADDIE; целевая аудитория: новые сотрудники;
  \item \textbf{Асинхронный учебный курс}~--- входной документ: учебная
    программа или~экспертные материалы; методология: SAM или~Backward~Design;
  \item \textbf{Compliance-курс}~--- входной документ: регламент или~политика;
    акцент на~точности (FactCheckAgent); методология~ADDIE.
\end{enumerate}

\subsection{Исследовательские гипотезы}

На~основании обзора литературы и~анализа предметной области
сформулированы три проверяемые гипотезы:

\textbf{H1 (продуктовая):} Мультиагентный конвейер ИИ-методиста
сокращает время создания структуры курса (time-to-draft) с~X~часов
до~Y~минут при~сохранении Bloom's-выравненности $\geq 70\%$.

\textbf{H2 (техническая):} Stateful мультиагентная архитектура
обеспечивает статистически значимо более высокое качество структуры
курса (faithfulness, Bloom's~coverage) по~сравнению с~single-prompt
baseline при~одинаковой базовой LLM.

\textbf{H3 (бизнесовая):} Unit-стоимость генерации одного курса
системой B4 составляет $\leq Z\%$ от~стоимости ручной разработки
у~внешнего подрядчика.

\subsection{Метрики проверки гипотез}

Для~каждой гипотезы определены метрики верификации:

\begin{itemize}
  \item H1: time-to-draft (мин.), Bloom's~coverage (\%),
    сравнение B4 vs.~B1;
  \item H2: faithfulness, Bloom's~coverage~--- критерий Вилкоксона
    (двусторонний, $\alpha = 0.05$), сравнение B4 vs.~B2;
  \item H3: unit-cost (руб./курс), сравнение с~рыночной ценой
    подрядчика.
\end{itemize}
