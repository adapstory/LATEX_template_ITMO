\chapter*{ВВЕДЕНИЕ}
\addcontentsline{toc}{chapter}{ВВЕДЕНИЕ}
\sloppy

Автоматизация разработки образовательных материалов на~основе больших языковых
моделей (LLM) является одним из~наиболее активно развивающихся направлений
на~пересечении машинного обучения и~EdTech.
По~оценкам аналитических агентств, мировой рынок корпоративного обучения
превысит \$51~млрд к~2030~году~\cite{edtech_market_report_2024},
при~этом главным сдерживающим фактором роста остаётся высокая стоимость
и~длительность создания качественного учебного контента.
Средний цикл разработки одного корпоративного курса силами методиста-эксперта
составляет от~нескольких дней до~нескольких недель и~требует специализированных
педагогических компетенций, которыми не~всегда обладают эксперты предметной области.

Существующие AI-инструменты для~автогенерации курсов (Coursebox~AI, iSpring~AI)
предоставляют one-shot генерацию на~основе единственного промпта,
не~поддерживают научно обоснованные методологии проектирования учебного процесса
(ADDIE, SAM, Backward~Design) и~ориентированы преимущественно на~англоязычный контент.
Применение мультиагентных LLM-конвейеров для~решения этой задачи
остаётся малоизученным: отсутствуют систематические сравнения архитектур
на~русскоязычном образовательном корпусе и~воспроизводимые бенчмарки
с~привязкой к~педагогическим метрикам (таксономия Блума).

\textbf{Объект исследования}~--- образовательная платформа Adapstory~(AI~LMS),
предоставляющая инструменты создания и~управления учебными курсами
для~B2B-клиентов.

\textbf{Предмет исследования}~--- микросервис ИИ-методист:
интеллектуальный методический ассистент, автоматизирующий создание,
упаковку и~актуализацию образовательных продуктов на~основе LLM
с~применением мультиагентной оркестрации.

\textbf{Цель работы}~--- разработка и~валидация интеллектуального
методического ассистента, обеспечивающего автоматизированное создание
структуры и~контента учебных курсов на~основе входных документов заказчика
с~сохранением педагогического качества, измеримого по~таксономии Блума.

Для~достижения цели поставлены следующие задачи:
\begin{enumerate}
  \item провести анализ предметной области: Customer~Development с~B2B-аудиторией,
    продуктовый обзор конкурентов и~систематический обзор научных подходов;
  \item оценить рыночный потенциал решения (TAM/SAM/SOM) и~сформулировать
    исследовательские гипотезы;
  \item спроектировать мультиагентную архитектуру конвейера генерации курсов
    и~обосновать выбор технологического стека через воспроизводимые эксперименты;
  \item реализовать систему: Document~Ingestion~Pipeline (Haystack~2.x),
    граф агентов LangGraph, LLM~Gateway, FastAPI-микросервис;
  \item провести валидацию системы: автоматизированную оценку (Ragas, Bloom's coverage),
    экспертный слепой тест (n~=~5--8~методистов) и~пилотное исследование с~B2B-клиентами;
  \item оценить бизнес-потенциал: unit-экономику, финансовую модель
    и~провести Customer~Development для~подтверждения Problem/Solution~Fit.
\end{enumerate}

\textbf{Научная новизна} работы состоит в~следующем:
предложена и~экспериментально верифицирована мультиагентная архитектура
генерации учебных курсов, в~которой впервые сочетаются
(1)~вариативная методологическая модель проектирования (ADDIE / SAM / Backward~Design)
как~переключаемый системный промпт,
(2)~stateful~DAG-оркестрация с~human-in-the-loop (LangGraph~StateGraph),
и~(3)~RAG-слой на~русскоязычном образовательном корпусе (Qdrant~+~deepvk/USER-bge-m3).
Показано, что данная архитектура превосходит single-prompt baseline
по~показателям faithfulness и~покрытия уровней таксономии Блума
при~статистически значимой разнице (критерий Вилкоксона, $p < 0.05$).

\textbf{Положения, выносимые на защиту:}
\begin{enumerate}
  \item Мультиагентный конвейер с~вариативной методологической моделью
    (ADDIE/SAM/Backward~Design) сокращает время создания структуры курса
    с~X~часов до~Y~минут при~сохранении Bloom's-выравненности $\geq 70\%$.
  \item Stateful~DAG-архитектура (LangGraph) обеспечивает статистически
    значимо более высокое качество структуры курса (faithfulness и~Bloom's~coverage)
    по~сравнению с~single-prompt подходом на~той же базовой LLM.
  \item Система прошла Problem/Solution~Fit: $N$~B2B-пилотов
    подтверждены письмами о~готовности от~ЛПР.
\end{enumerate}

Текущее состояние проекта: разработан и~протестирован MVP,
включающий Document~Ingestion~Pipeline и~агентный конвейер
с~методологией~ADDIE; проведены B2B-интервью и~получены LOI-письма
от~корпоративных заказчиков.

Структура работы: работа состоит из~введения, четырёх глав, заключения,
раздела об~использовании ИИ, списка использованных источников
и~десяти приложений.
Глава~1 содержит анализ предметной области и~постановку задачи.
Глава~2 описывает дизайн эксперимента и~обоснование выбора технологий.
Глава~3 описывает техническую реализацию системы.
Глава~4 содержит результаты валидации, анализ ограничений
и~бизнес-оценку.
